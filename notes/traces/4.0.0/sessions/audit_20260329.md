## Post-Mortem (Resolved Questions)

### Post 42701: "freeze watch" Google Trends (v4.0.0)
- **Forecast**: 80% Decreases, 12% Doesn't change, 8% Increases
- **Resolution**: Doesn't change (Mar 27 value within ±3 of Mar 17 value of 12)
- **Error type**: Missed key data
- **What happened**: The March 15 megastorm spike (100→51→12) was decaying as predicted, but a **separate freeze event** hit the central US on March 27-28 — freeze warnings issued across Kentucky, Indiana, Illinois, and Missouri. This second cold front kept Google Trends elevated near the start value.
- **Where reasoning diverged**: The agent assigned only 20% combined to "Doesn't change" + "Increases," reasoning "late March freeze events are rare." The 20% was correct in spirit but too low — March is still climatologically active for freeze events (the 2025-26 winter had repeated Arctic intrusions). The agent's summary flagged "No medium-range weather forecast API" as a capability gap.
- **What would have helped**: Open-Meteo API (free, 16-day forecasts) could have been queried, though at 10-day range (~50% accuracy) it likely wouldn't have caught this specific system. The bigger fix: a higher floor (25-35%) on weather-surprise probability during March.

### Post 42303: Slovenian Election Seat Difference (v3.5.0)
- **Forecast**: median +8.5 (SDS wins 9 more seats), CI [+1, +17]
- **Resolution**: -1 (GS won 29 seats, SDS won 28 — 0.68pp margin, smallest ever)
- **Error type**: Unforeseeable event + tool bug + narrow uncertainty
- **What happened**: Two factors drove the miss:
  1. **Black Cube spy scandal** broke March 17, fifteen days after the forecast. Leaked recordings linked Israeli intelligence firm Black Cube to SDS leader Janša. This flipped the election narrative — what initially appeared as anti-GS corruption evidence was reframed as foreign interference coordinated with SDS. Polls swung 4-9pp toward GS in the final week (Ninamedia: GS 29.4% vs SDS 25.7% by March 18).
  2. **Polymarket tool bug**: The `polymarket_price` tool returned an NBA game instead of the Slovenian election market. Polymarket actually had GS at ~60% to win most seats on March 2 — a strong contrarian signal against the poll consensus that the agent never saw.
  3. **Monte Carlo too narrow**: Used 3.5% polling std, far below Slovenia's historical error range (7-16pp in 2022).
- **Where reasoning diverged**: On March 2, the agent's poll-based forecast was defensible — all major polls showed SDS leading. But the Polymarket signal would have raised a major red flag, and wider uncertainty bands would have placed more probability mass at negative values.
- **What would have helped**: (1) Fixed `search_markets` tool (now available). (2) Historical polling error data for Slovenia would have widened the simulation bands. (3) Bidirectional sensitivity testing (now in prompts) would have tested sigma x 1.5.

### Post 42710: CP > 30% for US Negative GDP (v4.0.0)
- **Forecast**: 65% YES
- **Resolution**: No (CP stayed ≤30% on March 28)
- **Error type**: Overconfident (momentum extrapolation)
- **What happened**: The CP was exactly at 30% with +6pp recent momentum driven by the Iran conflict and oil shock. The agent extrapolated this momentum, predicting continued upward drift. Instead, the momentum stalled — the shock was absorbed and priced in. The agent's "information already priced in" factor was logit -0.5 at 0.6 confidence — too low for a scenario where the CP had already risen 6pp in 2 weeks on a single catalyst.
- **Where reasoning diverged**: Monte Carlo weighted "recent momentum (+3pp/week)" at 20% and "reduced momentum (+1.5pp/week)" at 40%. The "no drift" scenario (30%) and "reversal" (10%) were underweighted. CP is a bounded quantity with anchoring effects — sharp moves tend to revert more than random walks suggest.
- **What would have helped**: CP mean-reversion prior in the meta-prediction guidance. Current prompts warn "Don't over-fit small CP samples" but don't address momentum decay.

### Post 42726: ERIE Stock Price (v4.0.0)
- **Forecast**: 34% YES (price higher on Mar 28 vs Mar 20)
- **Resolution**: Yes (ERIE rose from $240.37 to ~$245+)
- **Error type**: Momentum extrapolation
- **What happened**: ERIE was in a severe downtrend (down 44.6% from 52-week high, near 52-week low). The agent used 30-day empirical drift (-0.62%/day) for Monte Carlo simulations, producing 27-34% probability. However, a **late-March market rebound** (S&P 500 soared on a Trump announcement March 23) reversed the downtrend. ERIE bounced from its oversold position.
- **Where reasoning diverged**: The agent correctly identified the oversold bounce possibility but weighted it at logit +0.5 / confidence 0.5 — too low given that ERIE was 1.17% above its 52-week low. The Monte Carlo's 30-day drift was dominated by the recent selloff regime and didn't account for market regime changes.
- **What would have helped**: The "macro regime shift check" now in HEAD prompts would have caught the March 23 market rebound if the forecast had been made later. More broadly, the conditional base rate (65% positive at this drawdown level, n=20) should have been weighted more heavily against the momentum-based Monte Carlo.

## What Has Been Fixed

- **Polymarket garbage data** (42303): The `polymarket_price` tool could return irrelevant markets (NBA game for Slovenian election). Fixed by `search_markets` — unified cross-platform prediction market search with LLM relevance filtering (v4.0.0+).
- **Polling uncertainty guidance** (42303): v3.5.0 had "Do not double-count uncertainty" (purely anti-widening). HEAD has bidirectional sensitivity testing ("scale sigma x 1.5, x 0.7") and "Do not add uncertainty you can't name" (requires naming missing uncertainties). This would have pushed the agent to test wider bands for Slovenia's historically volatile polling.
- **Stock trend extrapolation without macro check** (42726): HEAD adds "Macro regime shift check — search breaking news before computing drift estimates for stocks." This directly targets the failure of using stale 30-day drift during a market regime change.
- **Calibration philosophy** (all): "Consistency over brilliance" (avoid extremes) replaced by "Accuracy over caution" (both too-wide and too-narrow are wrong). Removes categorical limits on extreme probabilities.
- **Tool rate limiting** (pipeline): AskNews throttled proxy with retry, Google Trends retry with 60s backoff, market/FRED/arXiv throttles. Reduces tool failure cascades during tournament runs.
- **Reviewer restructured**: From evaluative ratings (1-5) to observational risk flags with categories (`overconfidence`, `computation_error`, `stale_data`). More likely to catch specific issues.
- **AskNews `search_google` hidden**: Prevents wasted calls on broken tool.

## What Remains Unaddressed

- **No weather forecast API** (42701): The agent's summary explicitly flagged "No medium-range weather forecast API" as a capability gap. Open-Meteo (free, no API key, 16-day forecasts, Python library) is the clear solution. Even at ~50% reliability for 10-day forecasts, it raises floor awareness of weather risk. For the freeze watch question, it would have alerted the agent that late March is still within freeze season and a weather tool was available to check.
  - **Recommendation**: Build `open_meteo` MCP tool. Query 16-day min temps at representative US locations for weather-dependent questions. Type: **tool/capability**.

- **CP momentum extrapolation** (42710, 42707): Agent over-extrapolated CP trends in both directions. In 42710, extrapolated +6pp/week momentum; in 42707, extrapolated -6pp/week momentum. Both trends reversed. The meta-prediction section warns "Don't over-fit small CP samples" but doesn't address momentum decay.
  - **Recommendation**: Add CP mean-reversion prior to meta-prediction guidance: "CP is a bounded quantity with anchoring bias. Sharp recent momentum shows diminishing returns — a +6pp week is more likely to produce a +2pp next week than another +6pp. Weight the 'no drift' scenario at ≥30% for any meta-prediction." Type: **principle**.

- **No historical polling error data** (42303): The agent used 3.5% polling std for a country with 7-16pp errors in 2022. No tool or prompt guidance to check historical polling accuracy before running election simulations.
  - **Recommendation**: Build polling error lookup — either a static dataset or a tool that retrieves country-specific historical polling accuracy from Wikipedia's opinion polling pages. For any election question, the agent should check the country's recent polling error magnitude before setting simulation parameters. Type: **tool/capability**.

- **Weather tail risk floor** (42701): Even with Open-Meteo, 10-day weather forecasts have ~50% reliability. For weather-dependent questions with 10+ day horizons during active weather seasons, the residual probability for "weather surprise" scenarios should use climatological base rates, not just the agent's subjective assessment.
  - **Recommendation**: Add principle to Google Trends/weather prompt section: "During November-April in the US, assign ≥25% residual probability to unexpected weather events in 10+ day forecast windows, regardless of current forecast." Type: **principle**.
