### 3. Subagent Usage: Still 0% in Live Sessions

| Context | Usage | Notes |
|---------|-------|-------|
| 10 recent live sessions | 0/10 (0%) | Agent always says "straightforward enough" |
| 8901 retrodict (v0.6.0) | YES | Used researcher + analyst, but tools limited in retrodict |

The 8901 retrodict is the only evidence of subagent activation. The agent's tool audit there notes: "The researcher couldn't access search tools effectively in retrodict mode. The analyst produced useful qualitative analysis but couldn't use FRED tools."

**Root cause:** The prompt says "If you have independent research or computation threads, use subagents — the parallelism saves time." This is permissive, not directive. The agent optimizes for its own convenience and always finds solo work "straightforward enough."

### 4. 429 Rate Limiting (Systemic Infrastructure Issue)

Every script that fetches individual posts triggers cascading 429s:

| Script | Posts Fetched | API Calls | 429 Retries Observed |
|--------|-------------|-----------|---------------------|
| feedback_collect.py | ~36 | ~72 (2 per post) | ~20 |
| resolution_update.py | ~36 | ~72 | ~18 |
| forecast_queue.py missed | ~23 | ~46 | ~15 |

**Root cause:** `AsyncMetaculusClient.fetch_post_json()` makes 1 JSON call, then checks for null fields and makes a 2nd HTML enrichment call. With Metaculus returning nulls for almost everything, virtually every call is doubled.

**Fix applied this session:** Serialized all scripts (forecast_queue.py, feedback_collect.py, resolution_update.py) with 2.0s base delay, 5.0s every 5 calls. forecast_queue.py switched from concurrent (semaphore=3) to sequential.
