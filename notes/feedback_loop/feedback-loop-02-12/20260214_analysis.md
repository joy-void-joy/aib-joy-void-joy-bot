#### MSFT EPS (41538): Every Version Failed

All versions forecast ~$3.85-3.91 EPS. Resolution: $5.16 (33% higher).

| Version | Median | CI Width | NE |
|---------|--------|----------|-----|
| v0.3.1  | $3.875 | $0.39 | 6.59 |
| v0.7.2  | $3.85 | $0.55 | 4.76 |
| v0.9.1  | $3.91 | $0.95 | 2.63 |
| v0.10.0 | $3.88 | $0.37 | 6.92 |

Root cause: All versions anchored on analyst consensus (~$3.81) without adequately modeling definitional ambiguity (GAAP vs non-GAAP). The v0.10.0 agent noted "GAAP vs non-GAAP ambiguity" in its meta-reflection but only assigned it -0.3 logits — insufficient.

v0.9.1 had the widest CI and lowest NE — wider was better here.

**"Trust the Monte Carlo" verdict:** The principle is correct for market-data-driven questions (stocks, yields, volatility). It fails for earnings because the Monte Carlo inputs themselves are wrong when there's definitional ambiguity. The fix is not to distrust Monte Carlo but to run *multiple* Monte Carlos under different interpretations and combine.

### Subagent Bug: Silent Empty Returns

**Found and fixed.** Two issues:

1. **Stream termination without ResultMessage:** When all subagent searches returned empty (common in retrodict mode), the SDK cancelled the stream. `composition.py` treated `result=None` as success with empty output. Fixed: now returns error or falls back to collected TextBlocks.

2. **TextBlocks not collected:** The subagent's intermediate reasoning was printed to display but never returned to the parent agent. Fixed: now collects all TextBlocks and uses them as fallback output.

v0.10.0 actively used subagents (5+ spawn_subagents calls in retrodictions), contradicting the "0% usage" finding from earlier sessions. The empty returns were a code bug, not a usage problem.

### Prompt Changes Made

1. **Decompose across ambiguity** (Core Principle): When definitional ambiguity detected, forecast under each interpretation separately, then combine as weighted mixture. Bridges gap between ambiguity detection (already in prompt) and proportional response (was missing).

2. **Structural vs scenario decomposition** (Subagents section): Two explicit patterns — structural decomposition uses `subquestion` agents, scenario decomposition uses `execute_code` in main thread.

3. **Scenario mixture** (NUMERIC_GUIDANCE approach #4): Added as 4th calibration approach alongside Monte Carlo, triangulation, and consensus+surprise.

## Meta-Level Findings

### Version Attribution Problem (Addressed)

v0.10.0 retrodictions used the pre-current prompt (before trust-computation, decomposition, consistency principles). We can't evaluate current prompt effectiveness from existing data.

**Fix:** Created `version_tag.py` script and retroactively tagged all versions v0.3.0-v0.12.0. Created CHANGELOG.md mapping versions to key behavioral changes.

### Bot Self-Assessment Quality

The agent's meta-reflections are honest and informative:
- Correctly identifies tool failures ("spawn_subagents returned empty output — tool failure")
- Notes GAAP/non-GAAP risk but doesn't act proportionally
- Provides good tool audits with error rates

**Gap:** The agent sees problems but its response is a logit adjustment, not structural. The decomposition principle should fix this by giving the agent a METHOD (run separate scenarios) instead of just a WARNING (note the risk).
