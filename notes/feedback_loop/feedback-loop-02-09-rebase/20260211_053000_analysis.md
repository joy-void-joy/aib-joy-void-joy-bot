### Pattern 1: get_cp_history is Broken — 100% Failure Rate

**The single most impactful finding.** Across ALL 14 retrodictions, `get_cp_history` fails with:
```
AttributeError: 'NoneType' object has no attribute 'get'
```

Root cause: The API returns `"aggregations": null` for certain questions. The chained `.get()` calls use default `{}` but `dict.get(key, default)` returns `None` when the key EXISTS with a null value — it only uses the default when the key is MISSING.

Fix: Replace `data.get("question", {}).get("aggregations", {}).get("recency_weighted", {})` with explicit `or {}` null-coalescing.

**Impact:** For meta-predictions (7/8 binary retrodictions), the agent cannot see CP trajectory. It repeatedly notes "could not access CP history — unable to verify whether recent trend is up or down." This forces pure first-principles reasoning about CP movement, which is clearly harder.

### Pattern 2: Binary Underconfidence on YES-Resolving Questions

The 4 YES-resolving meta-predictions got forecasts of 0.40, 0.50, 0.55, 0.55 (avg 0.50).
The 4 NO-resolving questions got 0.35, 0.35, 0.45, 0.35 (avg 0.375).

Separation is there (the agent is directionally right) but it's weak. The calibration diagram shows:
- 30-40% bucket: predicted 35%, actual 0% (2 forecasts — too few to be meaningful)
- 40-50% bucket: predicted 43%, actual 33% (3 forecasts)
- 50-60% bucket: predicted 53%, actual 100% (3 forecasts — **massive** underconfidence)

The 50-60% bucket is the most telling: when the agent said ~55% YES, it was RIGHT every time. This suggests the agent's moderate confidence is actually representing strong signals, but it's held back by excessive caution.

**Hypothesis:** The "Nothing Ever Happens" framework may be over-applied to meta-predictions. Meta-predictions about CP crossing a threshold are NOT about dramatic events — they're about statistical drift. The status-quo-is-sticky heuristic is less relevant when you're forecasting a random walk crossing a boundary.

### Pattern 3: No Subagent Usage in Any Retrodiction

Zero of 14 retrodictions spawned subagents. The agent handled everything in the main thread, using 10-39 tool calls directly. This is both a strength (low overhead, focused reasoning) and a potential weakness:

For 41521 (Venezuela), the agent made 39 tool calls and took 487 seconds. A deep-researcher subagent could have researched UNGA precedents while a link-explorer checked prediction markets simultaneously.

For simpler meta-predictions, main-thread is appropriate. The agent's tool usage is efficient:
- Average session: 125-240 seconds
- Average tool calls: 10-22
- Error rate: 5-20% (mostly get_cp_history failures)

### Pattern 5: Retrodict Search Quality is Poor

In retrodict mode, searches often return 0 results:
- Wayback Machine has no snapshots for many pages
- Exa search with date filtering drops most results (e.g., 4/5 dropped for 39672)
- web_search in retrodict returns empty for many queries

The agent compensates by relying more on:
- Direct question context from `get_metaculus_questions`
- General knowledge and reasoning
- polymarket/manifold prices (which work in retrodict)

This means retrodict reasoning is systematically shallower than live reasoning. It's still useful for calibration, but we should expect live forecasts to be better-informed.
