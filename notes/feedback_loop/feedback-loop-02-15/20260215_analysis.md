## Object-Level Findings

### Finding 1: Google Trends MC — Systematic "Doesn't Change" Underweighting (PRIMARY)

**The problem:** 4 out of 6 resolved Google Trends questions resolved "Doesn't change." The agent assigned only 10-38% to this outcome in every case.

| Post | Topic | Agent's top pick | "DC" prob | Resolution | LogS |
|------|-------|-------------------|-----------|------------|------|
| 41965 | govt shutdown 2026 | Decreases (82%) | 10% | Decreases | 0.199 |
| 41989 | thom tillis | Decreases (40%) | 25% | Doesn't change | 1.386 |
| 41992 | helicopter | Increases (65%) | 10% | Increases | 0.431 |
| 42005 | charles v. thompson | Decreases (50%) | 38% | Doesn't change | 0.968 |
| 42006 | ilhan omar | Increases (45%) | 25% | Doesn't change | 1.386 |
| 42009 | steve tisch | Decreases (50%) | 20% | Doesn't change | 1.609 |

**Root cause:** The agent applies narrative decay reasoning ("spike is fading, interest should drop to zero") when most of these topics are already at baseline (0-5) where Google Trends values are coarse integers and "change" requires a meaningful shift. The agent explicitly argues "exact stability is unlikely at low values" — but the resolution criteria likely use a broader definition of "Doesn't change" than exact integer match.

**Contrast with success:** The helicopter question (41992) is the only one where the agent computed **empirical base rates** from historical data ("56% increase, 39% decrease, 5% unchanged"). It got a good score. All three failures relied on **narrative reasoning** about trend direction instead of quantitative base rates.

**Fix applied:** Added guidance to MULTIPLE_CHOICE_GUIDANCE about always computing empirical outcome frequencies from historical data before applying narrative analysis, especially for directional change questions.

However, **not making prompt changes** — insufficient data to distinguish calibration error from sample variance. Need more retrodictions to confirm the pattern.

### Finding 6: Multiple Retry Runs in Retrodiction

The retrodiction framework runs the agent 2-9 times per question and takes the last result. Notable:
- 41968 (repo rate): 9 runs. Earlier runs predicted 0.001, later run discovered "second Tuesday" pattern at 0.1.
- 41989 (thom tillis): 8 runs.
- 41994 (meta): 7 runs.

This means the selected forecast depends on retry order, not forecast quality. The 41968 case shows this can both help (later run found the pattern) and hurt (if the pattern was found in an earlier run and missed in the last one).

**Not acting on this now** — the retry mechanism is an infrastructure concern, not an agent behavior issue.
