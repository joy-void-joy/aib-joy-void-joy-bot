## Object-Level Findings

### Root Cause: Macro Context Blindness (v3.5.0)

US-Israel strikes on Iran began Feb 28, 2026. All 79 v3.5.0 forecasts were made March 2-6.
The agent found Iran war info when it specifically searched (meta-predictions, Google Trends)
but **never searched for it** in financial forecasts. Stock and rate questions ran in isolated
sessions with Monte Carlo simulations anchored on historical volatility — missing the active
military conflict entirely.

**Impact:**
- 0/15 stock questions resolved YES (all fell with the market)
- 8/12 numeric outcomes exceeded agent's P90 (rates rose on risk-off)
- DJIA actual (47,417) was below agent's P10 (47,450)
- 90% CI coverage: 33% instead of 80%

The agent's Monte Carlo machinery is well-constructed (excellent results on 42376, 42400
when macro is stable). The inputs are the problem, not the computation.

### Reasoning Quality Assessment

**v3.5.0 Best (42372, Brier=0.001):** Near-perfect. Clear CP downtrend, agent resisted
reviewer pressure, parsed "military access" vs "sovereignty purchase" distinction.
When the agent has clear directional data and no macro blind spot, it's excellent.

**v3.5.0 Worst (42370, Brier=0.533):** Agent found extensive Iran war research but
treated a 2-day CP spike as a stable 10-point buffer. "The 10-point buffer above
threshold provides significant protection" — wrong when the buffer was earned in 48 hours
during a wartime spike.

**v3.5.0 Reviewer anti-pattern (42406, 42384):** Reviewer pushed stock probabilities
UP based on Monte Carlo output, making them MORE wrong. Agent deferred with "The reviewer
correctly points out I'm under-weighting my own quantitative evidence." The reviewer
can't question the Monte Carlo's assumptions — it reinforces whatever bias is embedded.

**v3.6.0 Process Quality:** Strong overall. Systematic multi-source triangulation in all
7 traces. CDF quality dramatically improved (42561 Brent crude has appropriate width with
4-scenario mixture). Reviewer working correctly (caught genuine error in 42511). Agent
recovers gracefully from tool failures (15 errors across 150 calls, no reasoning degradation).

### Reviewer Behavior

The reviewer is net-positive in v3.6.0 (caught real error in 42511) but was net-negative
in v3.5.0 stock forecasts (pushed probability in wrong direction 2/2 times). The issue:
the reviewer validates against the agent's own computation, not against external reality.
When the computation's inputs are wrong (missing macro context), the reviewer amplifies
the error.

window.** This is a general principle, not an Iran-specific patch.
