## Key Findings

### 1. Devtools Version Filter Bug (FIXED)

`scores show --version 4.0` used `resolve_version()` which falls back to all versions when exact match fails. `parse_semver("4.0")` returns None (requires X.Y.Z format), triggering immediate fallback. Added `match_versions()` for strict prefix filtering.

### 2. Opus Reviewer Grade Inflation (REDESIGNED)

82 summary.json files analyzed. Every single forecast scored 4-5/5 on all dimensions:
- Evidence quality: 58.5% at 5, 41.5% at 4. Zero below 4.
- Calibration sense: 90.2% at exactly 4. Almost no variance.
- 100% of summaries open with "Solid" or "Strong"
- 1.2% catch rate on real reasoning errors (1 catch across 82 reviews)

Useful parts (condensed_reasoning, tool_audit, capability_gaps) preserved. Scoring replaced with structured observations: ForecastClassification, RiskFlag, ReviewerEstimate.

### 3. Reflection Schema Validation Bug (FIXED)

In forecast 42915, the reflection tool failed 14 times before succeeding (~40% token waste). Root cause: agent passes `tentative_estimate` as a JSON string, Pydantic union validation rejects strings. Fix: `model_validator(mode="before")` that `json.loads()` strings.
