| Version | Net Effect | Details |

The reviewer validates against the agent's own computation, not external reality. When
computation inputs are wrong (stale macro data), the reviewer amplifies the error. This is
structural — fixing the inputs (macro context step) fixes the reviewer's incentive.

market-moving events from the last 7 days.** This is a general principle applicable to

incorporated Iran war). The v3.6.0 EUR/GBP trace proves it fails when the agent doesn't.

## Proposed Changes

### Priority 1: Add Macro Context Principle to Prompts
Add to Stock Direction Questions and Numeric Question Guidance sections:

> **Check for macro regime shifts.** Before running any simulation or computing drift
> estimates, search for major market-moving events from the last 7 days (geopolitical
> crises, policy shocks, market crashes). If a significant event has occurred since
> your historical data window, your empirical parameters may not reflect the current
> regime. Verify that today's price/rate is consistent with your data before simulating
> forward.

### Priority 2: Log Exa Response Body on Errors
Add `response.text` to the error logging in `exa.py` so future 400s are diagnosable.

### Priority 3: Fix bls_series Pandas Compatibility

### Priority 4: Suppress Wasted get_cp_history Calls
Add hint to tool description that CP history is unavailable for AIB tournament questions.
