### CRITICAL: CDF Generation Pipeline Distortion

The simulation produced a near-normal distribution: mean 122.6, std 2.9, range 110-131.
The submitted CDF has a fundamentally different shape:

| Region | Agent Intent | CDF Actual | Distortion |
|--------|-------------|------------|------------|
| < 95 pts | 0% | 5.04% | +5.04% fabricated |
| 95-118 | ~7% | 5.4% uniform | Shape destroyed |
| 119-126 | ~80% | 79.6% trapezoidal | Approximately preserved |
| 127-135 | ~3% | 4.9% uniform | Shape destroyed |
| > 135 pts | 0% | 5.04% | +5.04% fabricated |

**Root cause**: `NumericDistribution` compresses 100K simulation samples into 6 percentile points
(P10/P20/P40/P60/P80/P90), then linearly interpolates to the bounds. `_standardize_cdf` adds
minimum mass requirements. The result: ~10% probability mass in regions the simulation assigns
near-zero probability.

**Concrete impact**: Agent P10=119, CDF-implied P10=116.5 (2.5 points lower on a 40-point range).
For discrete questions with tight distributions relative to the question range, the pipeline
significantly misrepresents the agent's beliefs.

**Fix implemented**: Added optional P5/P95 tail anchor fields to `NumericForecast` in the
`cdf-fix` worktree. When the agent provides these (from simulation data), the CDF pipeline
uses them to anchor the tails instead of extrapolating from P10/P90 to the bounds.

Test results (42562 scenario):
| Metric | 6-point | 8-point | Improvement |
|--------|---------|---------|-------------|
| P10 error | 2.5 pts | 0.2 pts | 12.5x better |
| P95 error | 7.0 pts | 0.6 pts | 11.7x better |
| Fabricated tail mass | 10.1% | 5.1% | 50% reduction |

Branch: `cdf-fix`, commit: `f8470bd`.
